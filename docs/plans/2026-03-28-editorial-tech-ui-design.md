# Editorial-Tech UI 改版设计

## 目标
将前端从“视觉展示导向”改为“技术阅读导向”，同时提升整站交互与动效质量。

## 用户与约束
- 目标用户：技术博客读者
- 偏好：强表现动效
- 主题：深浅双模同质量
- 禁忌：企业后台风

## 核心改动
1. 全局样式
- 建立语义化 design token（背景、文本、边框、品牌色、阴影）
- 统一深浅主题变量映射
- 增加通用动效类（入场、分层、减少动效兼容）

2. 全局布局
- 顶部导航改为阅读导向信息结构
- 滚动状态下 Header 进入半实色玻璃态
- 页脚改为内容型文案，不做后台式信息条

3. 首页
- 首屏改为“读者起点”叙事
- 以最新主推文章 + 次级文章入口构建阅读路径
- 提供近期更新与高频主题标签

4. 文章归档页
- 增加关键词搜索、标签筛选、排序
- 以内容卡片网格强化可扫描性

5. 文章详情页
- 增加阅读进度条
- 左侧目录高亮当前章节
- 右侧同标签推荐与前后文章导航

## 交互原则
- 所有动效优先用于层次和反馈
- hover 联动仅限关键信息（标题、箭头、边框）
- 支持 prefers-reduced-motion

## 验收标准
- 用户可在 3 秒内定位主推文章入口
- 用户可在归档页通过关键词/标签找到目标文章
- 用户在详情页可感知阅读进度并快速跳转章节
- 深浅主题对比度和可读性一致
