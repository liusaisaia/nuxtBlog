# Build VoidZero-style Nuxt 4 Blog Plan

This plan outlines the steps to upgrade and complete the existing Nuxt project into a full-featured, high-performance blog with VoidZero's visual aesthetic.

## 1. Project Configuration & Toolchain Setup
- [ ] **Enable Nuxt 4 Compatibility**: Update `nuxt.config.ts` with `future: { compatibilityVersion: 4 }` to adopt latest features.
- [ ] **Integrate OXC**: Install `oxlint` and `eslint-plugin-oxlint` for high-performance linting.
- [ ] **Dependencies**: Ensure `pinia`, `@nuxtjs/tailwindcss`, `@nuxt/content`, `@nuxt/image`, `@nuxtjs/seo`, and `@nuxtjs/color-mode` are installed.
- [ ] **Directory Structure**: Organize into `app/` structure (if opting for Nuxt 4 defaults) or maintain standard `pages/`/`components/` with strict organization.

## 2. Visual Design (VoidZero Style)
- [ ] **Design System**: Refine `tailwind.config.ts` to match VoidZero's "Industrial Minimalist" look:
    - **Typography**: Inter (sans) + JetBrains Mono (code).
    - **Colors**: Deep blacks (`#0A0A0A`), subtle borders (`border-white/10`), high-contrast text.
    - **Effects**: Glassmorphism (backdrop-blur), fine 1px borders.
- [ ] **Global Layout**:
    - **Header**: Sticky, glass effect, with navigation and Theme Toggle.
    - **Footer**: Clean grid layout with social links and RSS.
    - **Transitions**: Subtle page and element transitions.

## 3. Core Features Implementation
- [ ] **Blog Engine (@nuxt/content)**:
    - Configure Markdown rendering with syntax highlighting (Shiki).
    - Implement **Article List** with pagination and lazy-loaded cover images.
    - Implement **Article Detail** with Table of Contents and "Prose" styling.
- [ ] **Taxonomy**: Create Tags/Categories system with dedicated pages.
- [ ] **Search**: Implement a "Cmd+K" command palette search using `useContentSearch` or flexsearch.
- [ ] **Comments**: Integrate Giscus (GitHub Discussions based) for a developer-friendly comment system.
- [ ] **RSS Feed**: Generate standard RSS/Atom feeds using `nuxt-feed` or server routes.

## 4. Performance & SEO
- [ ] **SEO**: Configure `useHead` for dynamic meta tags, OpenGraph images, and JSON-LD structured data.
- [ ] **Performance**:
    - Implement `@nuxt/image` for optimized formats (WebP/AVIF).
    - Verify Code Splitting (automatic in Nuxt/Vite).
    - Configure Caching rules (SWR/ISR) in `routeRules`.
- [ ] **Accessibility**: Audit colors for contrast and ensure keyboard navigation support.

## 5. Testing
- [ ] **Unit Testing**: Configure **Vitest** with `@vue/test-utils` and write tests for key components (e.g., `PostCard`, `ThemeToggle`).
- [ ] **E2E Testing**: Set up **Playwright** for critical user journeys (Navigation, Search, Theme Switch).

## 6. Verification
- [ ] Run full linting (`oxlint`).
- [ ] Run all tests (`vitest run`).
- [ ] Cross-browser check (Chrome, Safari, Firefox).
